# Feature Specification: Skill Deletion

**Feature Branch**: `007-skill-deletion`
**Created**: 2026-04-12
**Status**: Draft
**Input**: User description: 删除阶段提供两种模式——本地删除（仅移除本地和所有 Agent 中的 skill，不影响已发布版本）和完全删除（同时从本地和所有已发布平台移除）。完全删除是不可逆操作，需要明确确认。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Delete Skill Locally (Priority: P1)

用户删除本地 skill，从中央目录和所有 Agent 目录中移除。本地删除不影响已发布到云端的 skill——他人仍可安装使用。

**Why this priority**: 用户不再需要某个 skill 时应能轻松移除。本地删除是最常用的删除场景，且安全（不影响他人）。

**Independent Test**: 对一个已发布的 skill 执行本地删除，验证本地完全清除但云端仍可访问。

**Acceptance Scenarios**:

1. **Given** 本地存在一个已发布到 GitHub 的 skill，**When** 用户执行本地删除命令，**Then** 系统从本地中央目录移除该 skill，并通过同步机制清理所有 Agent 目录中的对应内容，GitHub 上的 skill 不受影响
2. **Given** 本地存在一个本地创建的 skill（未发布），**When** 用户执行本地删除命令，**Then** 系统从本地移除该 skill 并清理所有同步关联
3. **Given** 本地存在一个从内部平台安装的 skill，**When** 用户执行本地删除，**Then** 系统从本地移除，内部平台上的 skill 不受影响
4. **Given** 用户使用 `--dry-run` 模式，**When** 执行删除，**Then** 系统仅展示将被移除的目录列表和受影响的 Agent，不执行任何删除操作

---

### User Story 2 — Delete Skill Completely (Priority: P1)

用户完全删除一个 skill，同时从本地和所有已发布的平台移除。完全删除是不可逆操作，需要明确确认。

**Why this priority**: 某些场景下用户需要彻底移除一个 skill（如废弃的 skill），包括从云端平台清除。与本地删除同等重要，因为两者覆盖了不同的使用意图。

**Independent Test**: 对一个已发布到多个平台的 skill 执行完全删除，验证本地和所有平台上的 skill 均被移除。

**Acceptance Scenarios**:

1. **Given** 本地存在一个已发布到 GitHub 的 skill，**When** 用户执行完全删除命令并确认，**Then** 系统从本地移除 skill，并调用发布平台适配器从 GitHub 移除该 skill
2. **Given** skill 已发布到多个平台（GitHub + 内部平台），**When** 用户执行完全删除，**Then** 系统逐个从所有已发布平台移除，输出每个平台的删除结果
3. **Given** 完全删除中某个平台失败，**When** 系统继续执行，**Then** 已成功的平台保留删除结果，失败的平台输出错误信息，本地删除不受影响
4. **Given** 用户执行完全删除但未指定 skill 名称，**When** 系统无默认目标，**Then** 输出已安装的 skill 列表供用户选择
5. **Given** 用户使用 `-y` 模式（Agent 集成），**When** 执行完全删除，**Then** 系统跳过确认，直接执行删除

---

### User Story 3 — Preview Deletion Impact (Priority: P2)

删除前系统展示将受影响的位置列表（本地目录、Agent 目录、云端平台），用户确认后才执行。

**Why this priority**: 与其他生命周期阶段一致的安全要求——删除是不可逆操作，预览机制符合安全原则。

**Independent Test**: 执行删除命令，检查预览输出是否包含所有受影响的位置。

**Acceptance Scenarios**:

1. **Given** 用户执行本地删除，**When** 展示预览，**Then** 用户看到将被移除的本地目录和受影响的 Agent 列表
2. **Given** 用户执行完全删除，**When** 展示预览，**Then** 用户看到本地影响和所有将被移除的云端平台列表，系统明确提示"此操作不可逆"
3. **Given** 使用 `--dry-run` 模式，**When** 执行删除，**Then** 系统展示完整的预览信息，不执行任何删除操作

---

### Edge Cases

- 删除一个正在被 Agent 使用的 skill — 系统应执行删除，Agent 下次加载时自然发现 skill 不存在
- 完全删除时某个平台不可用（网络故障）— 系统应输出错误信息，已成功的平台保留删除结果，建议用户稍后手动清理失败平台
- 删除一个被其他 skill 依赖的 skill — 系统应提示依赖关系，但不阻止删除（由用户决定）
- 用户误删 skill 后想恢复 — 本地删除可通过重新安装恢复（如果已发布）；完全删除无法恢复
- skill 在本地有未发布的迭代修改 — 系统应提示存在未发布修改，确认后才执行删除

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST support deleting skills locally — removing from central directory and all synced Agent directories, without affecting published versions on any platform
- **FR-002**: System MUST support completely deleting skills — removing from local AND all published platforms via platform adapters
- **FR-003**: System MUST require explicit confirmation for complete deletion (affects cloud-published content)
- **FR-004**: System MUST show a preview of all affected locations (local directories, agents, cloud platforms) before executing deletion
- **FR-005**: System MUST support `--dry-run` mode that shows the deletion preview without executing
- **FR-006**: System MUST support `-y` mode for non-interactive execution (Agent integration)
- **FR-007**: System MUST delegate cloud deletion to the configured platform adapters — the core system handles orchestration only (remove local → invoke platform adapters → report results)
- **FR-008**: System MUST report per-platform results for multi-platform complete deletion (success/failure/skipped per platform)
- **FR-009**: System MUST warn when deleting a skill that has unpushed local modifications
- **FR-010**: System MUST output structured error messages with recovery hints when deletion operations fail
- **FR-011**: Local deletion MUST NOT require network access — it operates entirely on local file system

### Key Entities

- **Deletion Mode**: Two modes of deletion. "Local" removes the skill from the central directory and all synced Agent directories only. "Complete" additionally removes the skill from all published platforms via platform adapters.
- **Deletion Preview**: The summary of what will be removed before execution — local paths, affected agents, and affected cloud platforms. Generated by the facade before executing any deletion.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Local deletion removes a skill from all agents without affecting its published version on any platform
- **SC-002**: Complete deletion removes a skill from local storage and all published platforms with a single command
- **SC-003**: The `--dry-run` preview accurately lists all affected locations (local, agents, platforms)
- **SC-004**: Partial platform failure during complete deletion does not affect successfully deleted platforms — each platform result is independent
- **SC-005**: Local deletion completes without network access

## Assumptions

- Local deletion is the safe default — it only affects the user's local environment
- Complete deletion is destructive and irreversible — it removes the skill from platforms where other users may have installed it
- The deletion facade delegates cloud operations to the same platform adapters used by the publish phase (spec `005`) — no separate deletion adapter needed
- Platform adapters implement both publish and delete operations — the interface is unified per platform
- Local deletion does not require any platform authentication — it is a purely local operation
- Skill deletion removes the entire skill directory (including all files), not just the SKILL.md
